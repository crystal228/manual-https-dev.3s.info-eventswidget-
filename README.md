# Events Widget exploratory testing

Репозиторий содержит демонстрационный QA-отчет по исследовательскому тестированию страницы конструктора виджета мероприятий:

https://dev.3s.info/eventswidget/

## Подход к тестированию

Тестирование выполнялось вручную через браузер с использованием исследовательского подхода.

AI-инструменты использовались как инженерный помощник для:

- генерации идей для edge cases;
- анализа найденных проблем;
- структурирования QA-отчета;
- подготовки чек-листа и тест-кейсов;
- написания вспомогательного Playwright-скрипта для сбора диагностической информации:
  screenshots, console errors и failed network requests.

Вспомогательный Playwright-скрипт использовался для сбора диагностической информации:
скриншотов, ошибок консоли и неуспешных сетевых запросов. Скрипт не выполняет
автоматический поиск дефектов и не генерирует отчет.

Все дефекты, указанные в отчете, были подтверждены фактической проверкой страницы вручную.

## Что было протестировано

- Генерация preview виджета.
- Выбор тематики мероприятий.
- Выбор стран.
- Настройка ширины и высоты.
- Выбор цветовой темы, включая обязательную проверку красной темы.
- Корректность embed-кода.
- Адаптивность на desktop, tablet и mobile.
- UX/UI состояния формы.
- Browser console и network requests.
- Несколько edge-case сценариев для размеров и повторной генерации.

## Краткий итог

Ключевой сценарий генерации виджета работает нестабильно: для красной темы
embed-код генерируется без параметра `theme=red`, а preview после смены
настроек в ряде случаев остается старым или показывает состояние загрузки.

Это влияет на доверие к конструктору и может привести к установке
некорректного виджета на сайт пользователя.
